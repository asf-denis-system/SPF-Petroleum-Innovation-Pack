## Что это

Это **pack** (паспорт предметной области) — формализованное инженерное знание о **вторых принципах** управления нефтью как целевым продуктом upstream-производства.

**Главная идея:** Upstream — это не управление людьми и не управление пластом. Это управление **состояниями нефти** — от «запасы в пласте» до «товарная нефть на НПЗ». Пласт, скважины и инфраструктура — управляющие и обеспечивающие системы, а не цель.

**Модель трёх конвейеров:**
- **Конвейер нефти** (операционный) — сквозная цепочка состояний нефти: Запланирована → Происследована → Локализована → Подготовлена к доступу → Открыт доступ → Создано воздействие → Поднята на поверхность → Транспортирована → Подготовлена к сдаче → Поставлена на НПЗ
- **Конвейер проектов** (управленческий) — от регистрации БВ/ТВ до тиражирования решения
- **Конвейер технологий** (R&D) — от концепции до упакованного метода

Подробнее: [pack/04-work-products/03-three-pipeline-model.md](pack/04-work-products/03-three-pipeline-model.md)

## Ключевые понятия

| Термин | Определение |
|--------|-------------|
| **Вторые принципы** | Устойчивые физические и системные закономерности, которые:<br/>- Не зависят от оргструктуры<br/>- Не исчезают при смене подрядчика<br/>- Продолжают работать даже если о них забыли |
| **Нефть как целевая система** | Продукт, проходящий через сквозную цепочку состояний от пласта до НПЗ — именно он «застревает» на стыках |
| **Пласт** | Среда-носитель: хранит нефть, задаёт физические ограничения извлечения (управляющая система, не целевая) |
| **Состояние нефти** | Измеримый факт о том, в каком виде нефть находится сейчас (локализована, добыта, подготовлена и т.д.) |
| **Владелец стыка** | Роль, ответственная за переход нефти в следующее состояние — не исполнитель работ, а держатель Definition of Done |
| **БВ/ТВ** | Бизнес-возможность / технологическая возможность — узкое место на переходе конвейера нефти |
| **Pack** | Результат применения SPF к конкретной области (этот репозиторий) |
| **SPF** | Second Principles Framework — фреймворк для формализации вторых принципов |

## Ключевые вторые принципы в этом pack'е

### 🔴 Физические принципы
1. **Энергия конечна** — пластовая энергия исчерпаема; баланс энергии определяет схему разработки
2. **Флюиды текут по градиентам** — нельзя приказать воде не прорываться; можно изменить условия
3. **Пласт гетерогенен** — нет "среднего" пласта; есть зоны, барьеры, трещины
4. **Неопределённость неустранима** — подземное пространство наблюдается косвенно; полностью известным никогда не будет

### 🟠 Системные принципы
5. **Любое вмешательство меняет всю систему** — нет локального действия; всё связано давлением и потоками
6. **Запаздывание реакции** — пласт реагирует с задержкой (месяцы, годы)
7. **Система ищет новое равновесие** — после вмешательства пласт перестраивается

### 🟡 Вторые принципы инноваций
8. **Технология ≠ эффект** — результат зависит от состояния целевой системы, а не только от новизны технологии
9. **Лаборатория ≠ пласт** — масштаб имеет значение; лабораторные результаты часто не воспроизводятся в полевых условиях
10. **Инновация увеличивает вариативность** — новое ГТМ расширяет диапазон возможных исходов

### 🔵 Организационные принципы
11. **KPI меняет поведение системы** — метрика определяет выбор инженеров и траекторию разработки
12. **Информация теряется при передаче** — геолог → разработчик → технолог; в каждый переходе — упрощение и потери

## Как использовать этот pack

### Для геолога / модельера
- Научитесь распознавать гетерогенность (OA.PP.003)
- Используйте методику анализа неопределённости (UP.M.002)
- Избегайте ошибок history matching (FM.MOD.001)

### Для разработчика месторождения
- Проводите анализ энергетического баланса (OA.PP.001)
- Строите матрицы взаимовлияний (OA.SP.001)
- Планируете долгосрочно, учитывая запаздывание (OA.SP.002)

### Для инженера-технолога
- Мониторьте эффект ваших вмешательств (UP.M.005)
- Адаптируетесь к реальным данным (UP.M.006)
- Осознаёте, что вмешательство в одной скважине влияет везде (OA.SP.001)

### Для инженера инноваций
- Проверяйте, где технология работает (OA.IP.001)
- Квалифицируйте технологию перед масштабом (UP.M.007, UP.M.008)
- Понимаете разницу между лабором, моделью и полем (OA.IP.002)

### Для управляющего / руководителя
- Выбирайте KPI правильно (OA.OP.001)
- Балансируйте краткосроч vs. долгосроч цели (UP.M.009)
- Обеспечивайте информационный контур обратной связи (OA.OP.002)

## Статус разработки

> Lifecycle stages (FPF B.5.1): **Explore** → **Shape** → **Evidence** → **Operate**
>
> - **Explore** — исследование и первичная формулировка (гипотеза)
> - **Shape** — оформление в рабочий артефакт (проверено внутри)
> - **Evidence** — подтверждено на реальном объекте (ОПИ/ОПР)
> - **Operate** — применяется системно, тиражируется

## Дальнейший путь развития

1. **Подробные методические карточки** (03-methods/) — детальное описание каждого из 13 методов с примерами применения
2. **Дополнительные шаблоны рабочих продуктов** — матрица решений, план разработки, бланки квалификации
3. **Граф взаимосвязей** (07-map/) — визуализация связей между вторыми принципами, методами и failure modes
4. **Интеграция с актуальными примерами** — кейсы из реальных месторождений (конечно, с обезличиванием)


SPF **универсален по форме и процессу**, но **не по содержанию**:
- Структура pack'а одинакова для любых областей
- Процесс создания знания одинаков
- Lint и гейты одинаковы
- Но наполнение pack'а всегда доменно-специфично
