# Distinctions: Upstream Petroleum Development

Ключевые различения (concepts), которые структурируют эту область.

**Уровни уверенности (Confidence Levels):**
| Уровень | Значение |
|---------|----------|
| **L0** | Теоретическая гипотеза — не проверена на практике |
| **L1** | Подтверждено моделью или лабораторией |
| **L2** | Подтверждено в поле (единичные кейсы) |
| **L3** | Отраслевой стандарт — многократно воспроизведено на разных объектах |

## [D.001] Целевая система vs. Управляющие системы

**Definition:** Целевая система — **нефть** (продукт, который должен пройти сквозную цепочку состояний от пласта до НПЗ). Управляющие системы — пласт, скважины, ГТМ, инфраструктура — всё, что обеспечивает переходы нефти между состояниями.

**Contrast with:** Считать целевой системой пласт или скважину — это приводит к оптимизации «носителя», а не «продукта». Можно иметь отличный пласт и плохо управляемую добычу.

| Целевая система (нефть) | vs. | Управляющие/обеспечивающие системы |
|------------|-----|------------|
| Проходит состояния: запланирована → локализована → добыта → поставлена | | Обеспечивают переходы нефти между состояниями |
| Её состояние = результат для бизнеса | | Их состояние = средство, не цель |
| Застревает на стыках между функциями | | Каждая оптимизируется локально |
| Измеряется на каждом переходе (дебит, качество, потери) | | Измеряются по своим KPI (проходка, закачка, расход) |

**Пласт в новой модели:** Пласт — среда-носитель нефти. Он задаёт физические ограничения (энергия, гетерогенность, неопределённость), которые влияют на скорость и возможность перехода нефти в следующее состояние. Пласт — объект *инженерного познания*, нефть — объект *управления производством*.

**Why it matters:** Если целевая система — пласт, оптимизируем геологию. Если целевая — нефть, оптимизируем весь путь: от «где запасы» до «где товар». Это меняет KPI, роли и то, что считается «застреванием».

**Related items:**
- Methods: UP.M.001 (Анализ состояния целевой системы)
- Work Products: [WP.ThreePipelines — Конвейер нефти](../04-work-products/03-three-pipeline-model.md)
- Distinctions: D.010 (Состояние нефти vs. Процесс добычи)

**Confidence:** L2 — подтверждено в практике управления добычей; отдельные компании применяют value-stream подход к upstream.

**SoTA**: `current`
- Revision criterion: Граница применимости возникает на стыке пласт↔скважина, когда физика пласта напрямую определяет состояние нефти без отдельного управленческого перехода.

## [D.002] Модель vs. Реальность

**Definition:** Модель (ГДМ) — это формализованная гипотеза о поведении пласта. Реальность — это поведение пласта, которое наблюдается.

**Contrast with:** Путаница: видеть модель как отражение реальности; верить калибровке как доказательству правильности.

| Модель | vs. | Реальность |
|--------|-----|-----------|
| Набор параметров и уравнений | | Поле давлений, насыщения, потоков |
| Может быть отлично откалибрована в прошлом | | Будущее может пойти по другой ветке |
| Подчинена выбору сеточки, параметризации | | Не подчинена нашим параметризациям |

**Why it matters:** История matching маскирует неопределённость. Модель, хорошо сошедшаяся к прошлому, может плохо предсказывать будущее.

**Related items:**
- Failure Mode: FM.G.001 (Путаница между калибровкой и валидацией)
- SoTA: SOTA.M.001

**Confidence:** L3 — фундаментальный принцип reservoir engineering; описан в стандартах SPE и учебниках.

**SoTA**: `current`
- Revision criterion: Модель становится ближе к реальности, если она предсказывает поведение в новых сценариях.

## [D.003] Детерминированное vs. Вероятностное

**Definition:** Детерминированное решение игнорирует неопределённость входных данных. Вероятностное решение учитывает её.

**Contrast with:** Видеть неопределённость как ошибку измерения, которую можно устранить, если приложить усилия.

| Детерминированный подход | vs. | Вероятностный подход |
|----------|-----|-------------|
| "Это правильное решение" | | "Это оптимальное при неопределённости" |
| Один расчёт | | Портфель сценариев |
| Риск — ошибка расчёта | | Риск — фундаментальное свойство подземного пространства |

**Why it matters:** Только вероятностный подход учитывает фундаментальную неопределённость подземного пространства.

**Related items:**
- Methods: UP.M.002 (Анализ сценариев)
- Failure Mode: FM.ORG.001 (Решение принято детерминированно, без учёта неопределённости)

**Confidence:** L3 — отраслевой стандарт (SPE PRMS, стохастическое моделирование запасов).

**SoTA**: `current`
- Revision criterion: Вероятностный подход валиден, когда история достаточна для построения распределения.

## [D.004] Локальное vs. Системное

**Definition:** Локальное действие — это вмешательство, которое предполагается воздействовать только локально. Системное действие — это признание, что любое вмешательство меняет всю систему.

**Contrast with:** Видеть скважину как изолированный объект; думать, что ГРП не влияет на давление в соседних скважинах.

| Локальное мышление | vs. | Системное мышление |
|--------|-----|-------------|
| "Изменим режим одной скважины" | | "Изменим распределение давлений во всем месторождении" |
| Оптимизируем одну скважину | | Оптимизируем портфель всех действий |
| Скважины независимы | | Скважины связаны давлением и потоками |

**Why it matters:** Локальное мышление приводит к субоптимальным портфелям. Действие в одной скважине всегда влияет на давление и потоки везде.

**Related items:**
- Failure Mode: FM.SYS.001 (Вмешательство в одну зону создало катастрофу в другой)
- Methods: UP.M.003 (Матрица взаимовлияний)

**Confidence:** L3 — доказано численным моделированием и промысловой практикой интерференции скважин.

**SoTA**: `current`
- Revision criterion: Локальное становится адекватным, если расстояние большое или проницаемость очень низкая (граница применимости).

## [D.005] Вмешательство vs. Наблюдение

**Definition:** Вмешательство — это активное действие, которое меняет поле параметров системы (ППД, ГРП, скважина). Наблюдение — это пассивное измерение, которое не меняет систему.

**Contrast with:** Путаница: скважина — наблюдение (она не меняет пласт), или избыточное вмешательство там, где достаточно наблюдения.

| Вмешательство | vs. | Наблюдение |
|---------|-----|-----------|
| Бурение, закачка, ГРП | | Измерители давления, пробоотборники |
| Меняет состояние системы | | Не меняет состояние системы |
| Всегда имеет стоимость | | Имеет стоимость, но не меняет систему |
| Требует обратимости анализа | | Безопасно для экспериментов |

**Why it matters:** Оптимальный портфель часто требует баланса между вмешательствами (разработка) и наблюдениями (мониторинг).

**Related items:**
- Work Product: Матрица "Вмешательство vs. Наблюдение"

**Confidence:** L3 — базовый принцип экспериментального дизайна в разработке месторождений.

**SoTA**: `current`
- Revision criterion: Граница размывается, когда скважина используется и для вмешательства, и для наблюдения.

## [D.006] Запаздывание vs. Мгновенный отклик

**Definition:** Запаздывание — это задержка реакции пласта на вмешательство. Мгновенный отклик — это предположение, что система реагирует сразу.

**Contrast with:** Видеть закачку сегодня как изменение дебита завтра (игнорирование запаздывания).

| Запаздывание | vs. | Мгновенный отклик |
|----------|-----|----------|
| Эффект ППД через месяцы–годы | | Предполаганас мгновенная реакция |
| Нужно планировать долгосрочно | | "Кран повернул — результат сейчас" |
| Система инерционна | | Система безынерционна |

**Why it matters:** Запаздывание часто приводит к завышенным ожиданиям от проектов в первые месяцы и удивлениям в последующие годы.

**Related items:**
- Failure Mode: FM.SYS.002 (Проект казался неудачным, пока эффект не проявился)
- Methods: UP.M.004 (Анализ временных рядов с выделением лага)

**Confidence:** L3 — подтверждено теорией пьезопроводности и многолетней промысловой практикой ППД.

**SoTA**: `current`
- Revision criterion: Наблюдаемый лаг между воздействием и откликом.

## [D.007] Баланс энергии vs. Игнорирование энергетики

**Definition:** Баланс энергии — это осознание, что пластовая энергия конечна и её траты определяют режим разработки. Игнорирование — это видеть только дебит.

**Contrast with:** Оптимизировать добычу по дебиту, не считая энергии (жечь энергию впустую).

| Энергетический баланс | vs. | Игнорирование энергии |
|---------|-----|---------|
| "Сколько энергии осталось?" | | "Сколько нефти мы качаем сегодня?" |
| Долгосрочный горизонт | | Краткосрочный горизонт |
| КИН определяется энергией | | КИН — чёрный ящик |

**Why it matters:** Если не следить за балансом энергии, месторождение падает в добыче быстрее, чем геологически неизбежно.

**Related items:**
- Failure Mode: FM.PHYS.001 (Месторождение упало в добыче, потому что энергия была потрачена впустую)

**Confidence:** L3 — фундаментальный закон термодинамики; основа материального баланса в разработке.

**SoTA**: `current`
- Revision criterion: Наблюдаемое снижение давления и дебита.

## [D.008] Гетерогенность vs. Гомогенный пласт

**Definition:** Гетерогенность — признание, что пласт неоднороден (зоны проницаемости, барьеры, трещины). Гомогенность — упрощающее предположение об однородности.

**Contrast with:** Видеть пласт как однородный объект; применять средние параметры везде.

| Гетерогенный пласт | vs. | Гомогенный пласт |
|---------|-----|---------|
| "Зоны высокой и низкой проницаемости" | | "Пласт одинаков везде" |
| Разные скважины ведут себя по-разному | | Все скважины ведут себя одинаково |
| Требует детального моделирования | | ГДМ на сетке 50×50 км |

**Why it matters:** Гомогенное предположение приводит к неправильным прогнозам добычи и неожиданным режимам в полевых скважинах.

**Related items:**
- Failure Mode: FM.PHYS.002 (Внезапный прорыв воды, потому что забыли про трещины)
- Methods: UP.M.005 (Анализ гетерогенности)

**Confidence:** L3 — доказано геостатистикой, керновыми исследованиями и промысловыми данными.

**SoTA**: `current`
- Revision criterion: Сейсмика, геологическое описание, данные скважин.

## [D.009] Конвейер нефти vs. Конвейер проектов vs. Конвейер технологий

**Definition:** Три конвейера — это три разных потока объектов с разными предметами управления. Конвейер нефти — операционный (объект: нефть/запасы в состоянии). Конвейер проектов — управленческий (объект: проект как контейнер изменения). Конвейер технологий — R&D (объект: технология/метод как продукт).

**Contrast with:** Видеть единый «проект разработки», в котором смешаны физическая добыча, управление изменениями и R&D — это приводит к размытию ответственности и потере управляемости.

| Конвейер нефти | vs. | Конвейер проектов | vs. | Конвейер технологий |
|-----------|-----|-----------|-----|-----------|
| Объект: поток нефти в состоянии | | Объект: проект (от БВ/ТВ до тиража) | | Объект: технология/метод как продукт |
| Операционный ритм | | Проектный ритм | | R&D ритм |
| Стоп-сигнал: нефть «застряла» | | Стоп-сигнал: нет пакета внедрения | | Стоп-сигнал: нет границ применимости |
| Владелец: промысел/разработка | | Владелец: PM/портфельный комитет | | Владелец: продукт-оунер технологии |

**Why it matters:** Если не разделять конвейеры, ответственность за «застревание» нефти размывается: каждый участник делает «своё», не видя всей цепочки.

**Related items:**
- Work Products: [WP.ThreePipelines](../04-work-products/03-three-pipeline-model.md)
- Bounded Context: Модель трёх конвейеров (01A)

**Confidence:** L1 — концептуальная модель, подтверждена логикой системного анализа; полевые кейсы в накоплении.

**SoTA**: `current`
- Revision criterion: Граница применимости возникает, когда конвейеры работают синхронно и разделение условно.

## [D.010] Состояние нефти vs. Процесс добычи

**Definition:** Состояние нефти — это измеримый факт о том, в каком виде нефть находится сейчас («локализована», «открыт доступ», «поднята на поверхность»). Процесс добычи — это действия, которые осуществляют переход между состояниями (бурение, ГТМ, ППД).

**Contrast with:** Управлять процессами (бурение выполнено, ГТМ проведено), не проверяя, что нефть реально перешла в следующее состояние.

| Состояние нефти | vs. | Процесс добычи |
|---------|-----|---------|
| Факт о нефти (измеримый) | | Действие команды (выполняемое) |
| «Нефть локализована» — есть запасы с точностью достаточной для решений | | «Исследования выполнены» — работы сделаны |
| Переход = Definition of Done | | Выполнение = отчёт о работах |
| Управляется через владельца стыка | | Управляется через исполнителя работ |

**Why it matters:** Можно выполнить все работы (процессы) и при этом нефть не перейти в следующее состояние — если на стыке не было доказательств готовности. Управление через состояния, а не процессы, делает переходы явными.

**Related items:**
- Roles: R.006 (Владелец стыка)
- Work Products: [WP.ThreePipelines — Конвейер нефти](../04-work-products/03-three-pipeline-model.md)

**Confidence:** L2 — подтверждено в практике Agile/Kanban управления разработкой ПО; применение в upstream — в накоплении.

**SoTA**: `current`
- Revision criterion: Применимо, когда переходы между состояниями явно измеримы. При высокой неопределённости границы состояний размыты.
