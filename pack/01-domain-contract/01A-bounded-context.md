# Bounded Context: Upstream Petroleum Development

---

## Определение контекста

**Название:** Управление состоянием нефти как целевого продукта upstream-производства

(English: Oil State Management in Upstream Petroleum Production)

**Область применимости:** Разработка углеводородных месторождений (нефть, газ) в инженерном и операционном масштабе
**Тип работы:** Системно-инженерное описание (целевая система — нефть; пласт, скважины, инфраструктура — управляющие и обеспечивающие системы)

---

## Объект описания

**Целевая система:** Нефть — как продукт, проходящий через сквозную цепочку состояний от «запасы в пласте» до «товарная нефть на НПЗ»

| Что это IS | Что это IS NOT |
|-----------|---------------|
| Объект с измеримыми состояниями (насыщение, доступность, подвижность, качество) | Абстрактный «ресурс недр» |
| Продукт, проходящий через стадии (локализована → добыта → подготовлена) | Просто «объём в плане» |
| Объект, застревающий на стыках между функциями | Постоянно текущий поток |
| Предмет управления всех трёх конвейеров | Побочный результат бурения |
| Измеримый на каждом переходе (дебит, качество, потери) | Неизмеримый до сдачи на НПЗ |

**Управляющие и обеспечивающие системы** (не целевые):

| Система | Роль по отношению к нефти |
|---------|--------------------------|
| **Пласт** | Среда-носитель: хранит нефть, определяет условия и ограничения извлечения |
| **Скважины** | Каналы доступа: обеспечивают физический переход «пласт → поверхность» |
| **ППД/ГТМ** | Управляющие воздействия: изменяют условия для перехода нефти в следующее состояние |
| **Инфраструктура сбора/подготовки** | Транспортно-технологическая цепь: обеспечивает переход «поверхность → товар» |

---

## Модель трёх конвейеров

Управление upstream-производством описывается через три взаимосвязанных конвейера. Каждый конвейер — это поток объектов, меняющих своё состояние; управление строится через контроль переходов между состояниями.

### Конвейер 1: Целевая система «НЕФТЬ» (операционный)

Предмет управления — **нефть/запасы нефти в нужном состоянии**, а не процессы добычи.

Сквозная цепочка состояний нефти:

```
Запланирована → Происследована → Локализована → Подготовлена к добыче
→ Открыт доступ → Создано воздействие → Поднята на поверхность → Поставлена на НПЗ
```

Ключевая идея: проблемы возникают **на стыках** между состояниями, и именно там «застревает нефть». У каждого перехода должен быть явный **владелец стыка** — ответственная роль за прохождение перехода.

### Конвейер 2: Проекты (управленческий)

Предмет управления — **проект как контейнер изменения**: начинается с проблемы (БВ/ТВ), заканчивается тиражируемым результатом.

Цепочка стадий проекта:

```
Регистрация БВ/ТВ → Приоритизация → Подбор решений → Инициация
→ Реализация → Испытания (ОПИ/ОПР) → Упаковка внедрения → Тиражирование
```

Каждая стадия имеет явный выход (артефакт) и gate-решение (GO / NO-GO / PIVOT).

### Конвейер 3: Технологии (R&D/НИОКР)

Предмет управления — **технология/метод как продукт** (не проект). Конвейер технологий работает внутри конвейера проектов.

Цепочка зрелости технологии:

```
Концепция → Обоснование/Модель → Прототип → Образец для объекта
→ Испытание на объекте → Упаковка продукта → Поддержка и улучшения
```

### Связь конвейеров

- Узкое место в конвейере нефти → регистрируется как БВ/ТВ → входит в конвейер проектов.
- Конвейер проектов → заказывает технологию → конвейер технологий её дозревает.
- Готовая технология → возвращается в конвейер нефти как новый метод.

**Полная модель:** см. [WP.ThreePipelines](../04-work-products/03-three-pipeline-model.md).

## Инварианты (Invariants)

Правила, которые **никогда не должны нарушаться** в этом bounded context. Нарушение инварианта означает, что решение принято вне рамок этой модели.

| # | Инвариант | Следствие нарушения |
|---|-----------|-------------------|
| **INV.001** | Переход нефти в следующее состояние не засчитывается без артефакта-доказательства | Нефть «виртуально» переходит, но физически застряла — потери невидимы |
| **INV.002** | Технология не тиражируется без подтверждённого эффекта на объекте (ОПИ/ОПР) | Масштабирование неподтверждённого метода → системные потери |
| **INV.003** | Добыча не планируется без оценки энергетического баланса пласта | Краткосрочная оптимизация уничтожает долгосрочный КИН |
| **INV.004** | Технология не применяется за пределами квалифицированных условий применимости | Копирование «успешного» метода в другую геологию → провал |
| **INV.005** | Gate-решение (GO/NO-GO) не принимается без baseline потерь | Нечем измерить, сработало ли решение |
| **INV.006** | Каждый переход конвейера нефти имеет явного владельца стыка | Ответственность за «застревание» размывается |

## Glossary (Словарь контекста)

Термины, специфичные для этого bounded context. Термины из других контекстов могут означать другое.

| Термин | Определение в этом контексте |
|--------|------------------------------|
| **Нефть** | Целевой продукт — материальный объект, проходящий состояния от «в пласте» до «товар на НПЗ» |
| **Состояние нефти** | Измеримый факт о текущей стадии нефти (не процесс, не план — факт) |
| **Пласт** | Среда-носитель нефти; управляющая система, не целевая |
| **Переход** | Смена состояния нефти, подтверждённая артефактом-доказательством |
| **Стык** | Граница между двумя состояниями нефти, где возникают потери и размытие ответственности |
| **Владелец стыка** | Роль, отвечающая за засчитывание перехода нефти (не за выполнение работ) |
| **БВ/ТВ** | Бизнес-возможность / Технологическая возможность — зафиксированное узкое место на конкретном стыке |
| **Baseline** | Измеренные потери (нефти/денег/времени) на стыке до вмешательства; обязателен для gate-решения |
| **ИЛП** | Инструкция — пакет внедрения, позволяющий тиражировать метод без «героев» |
| **ОПИ/ОПР** | Опытно-промысловые испытания/работы — обязательный этап перед тиражированием |
| **Gate-решение** | GO / NO-GO / PIVOT — явное решение на границе стадий конвейера проектов |

## U.Systems и U.Episteme

Разделение между **операционными системами** (что существует в реальности) и **моделями знания** (что мы знаем/думаем об этой реальности).

### U.Systems (операционные объекты)

| Система | Тип | Пример |
|---------|-----|--------|
| Нефть в состоянии | Целевая система | «2 млн т локализованных запасов в зоне X» |
| Пласт | Среда/носитель | Коллектор Ю1, давление 250 атм, насыщение 0.7 |
| Скважина | Управляющая система | Скв. 1234, ГС, L=800м, дебит 80 т/сут |
| ППД-система | Управляющая система | 5 нагнетательных скважин, закачка 3000 м³/сут |
| Инфраструктура | Обеспечивающая система | Трубопровод DN300, УПН мощность 1500 т/сут |
| Проект | Контейнер изменения | Проект «Повышение охвата ГРП», статус: ОПИ |
| Технология/метод | Продукт R&D | «Метод подбора кандидатов на ГРП по признаку X», TRL 5 |

### U.Episteme (модели и знание)

| Модель | Тип | Что описывает |
|--------|-----|---------------|
| ГДМ | Гипотеза о пласте | Поведение пласта при разных сценариях разработки |
| План разработки | Управленческая модель | Целевые состояния нефти и методы их достижения |
| Карточка БВ/ТВ | Описание проблемы | Узкое место, baseline потерь, привязка к стыку |
| Паспорт проекта | Описание решения | Объект испытаний, план, бюджет, критерии GO |
| Матрица квалификации технологии | Границы применимости | При каких условиях метод работает/не работает |
| Протокол эффекта (ОПИ) | Доказательство | Факт vs. план, вариативность, выводы |
