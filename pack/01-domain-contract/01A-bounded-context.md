# Bounded Context: Upstream Petroleum Development

---

## Определение контекста

**Название:** Управление состоянием пластовой системы в разработке месторождения

(English: Reservoir System State Management in Petroleum Field Development)

**Область применимости:** Разработка углеводородных месторождений (нефть, газ) в инженерном масштабе  
**Тип работы:** Системно-физическое описание (не психология, не организационный менеджмент, не экономика)

---

## Объект описания

**Целевая система:** Пласт → скважина → флюиды → энергия → инфраструктура → поток продукции

| Что это IS | Что это IS NOT |
|-----------|---------------|
| Система с измеримыми состояниями (давление, насыщение, энергия) | Просто набор скважин |
| Объект, реагирующий на вмешательства предсказуемо (в среднем) | Чёрный ящик |
| Носитель физических законов (энергетика, гидродинамика) | Объект управления людьми |
| Система, стремящаяся к новому равновесию | Статичная система |
| Объект наблюдения (косвенное, через редкие точки, через шум) | Полностью известная система |

## Модель трёх конвейеров

Управление upstream-производством описывается через три взаимосвязанных конвейера. Каждый конвейер — это поток объектов, меняющих своё состояние; управление строится через контроль переходов между состояниями.

### Конвейер 1: Целевая система «НЕФТЬ» (операционный)

Предмет управления — **нефть/запасы нефти в нужном состоянии**, а не процессы добычи.

Сквозная цепочка состояний нефти:

```
Запланирована → Происследована → Локализована → Подготовлена к добыче
→ Открыт доступ → Создано воздействие → Поднята на поверхность → Поставлена на НПЗ
```

Ключевая идея: проблемы возникают **на стыках** между состояниями, и именно там «застревает нефть». У каждого перехода должен быть явный **владелец стыка** — ответственная роль за прохождение перехода.

### Конвейер 2: Проекты (управленческий)

Предмет управления — **проект как контейнер изменения**: начинается с проблемы (БВ/ТВ), заканчивается тиражируемым результатом.

Цепочка стадий проекта:

```
Регистрация БВ/ТВ → Приоритизация → Подбор решений → Инициация
→ Реализация → Испытания (ОПИ/ОПР) → Упаковка внедрения → Тиражирование
```

Каждая стадия имеет явный выход (артефакт) и gate-решение (GO / NO-GO / PIVOT).

### Конвейер 3: Технологии (R&D/НИОКР)

Предмет управления — **технология/метод как продукт** (не проект). Конвейер технологий работает внутри конвейера проектов.

Цепочка зрелости технологии:

```
Концепция → Обоснование/Модель → Прототип → Образец для объекта
→ Испытание на объекте → Упаковка продукта → Поддержка и улучшения
```

### Связь конвейеров

- Узкое место в конвейере нефти → регистрируется как БВ/ТВ → входит в конвейер проектов.
- Конвейер проектов → заказывает технологию → конвейер технологий её дозревает.
- Готовая технология → возвращается в конвейер нефти как новый метод.
