# Register Bridge: Physical ↔ Economic

**Pack:** upstream-petroleum
**FPF Reference:** B.1.6 — Δ_work additivity across holonic boundaries
**Назначение:** Декларация ресурсного базиса и явных формул-мостов между физическим (технологическим) и экономическим регистрами.

> **Ключевой принцип:** Мерить в физическом регистре. Считать деньги — в экономическом. Мост — явная версионированная формула с задокументированными допущениями. Никогда не смешивать регистры в одном показателе без моста.
>
> Нарушение → FM.EW.001

---

## 1. Два регистра

| Регистр | Что содержит | Зависит от | Меняется при смене |
|---------|-------------|------------|-------------------|
| **Физический** | Удельные расходы ресурсов на переход нефти | Физических условий (обводнённость, давление, КПД оборудования) | Изменения физических условий или оборудования |
| **Экономический** | Затраты в денежном выражении | Физики + тарифов, нормативов, цен | Изменения тарифов/цен (даже без изменения физики) |

**Диагностическая ценность разделения:**
- «Затраты выросли — это тариф вырос или КПД упал?» — без разделения невозможно ответить быстро.
- Физические параметры стабильны при сравнении между скважинами и месторождениями; экономические — нет.

---

## 2. Ресурсный базис (Resource Basis Declaration)

Полный список типов ресурсов, потребляемых на переходах конвейера нефти. Суммирование через Δ_work корректно только при явном объявлении базиса.

| Тип ресурса | Код | Единица физического регистра | Единица экономического регистра | Переходы конвейера |
|-------------|-----|------------------------------|----------------------------------|--------------------|
| Электроэнергия | RES.EL | кВт·ч / т.жидкости | руб / т.нефти | Подъём, перекачка, подготовка |
| Тепловая энергия | RES.TH | Гкал / т.жидкости | руб / т.нефти | Подготовка нефти, нагрев |
| Реагенты (хим.) | RES.CH | кг / т.жидкости | руб / т.нефти | Подготовка, ингибирование |
| Потери нефти | RES.LO | % от добычи (т) | руб / т.нефти | Подготовка, транспорт |
| Вода закачки (ППД) | RES.WI | м³ / т.нефти | руб / т.нефти | Поддержание давления |
| Деградация оборудования | RES.DG | η_pump (%), MTBF (сут) — *физические прокси* | руб / т.нефти (Amort_pump, Amort_UPN) — *только эконом.* | Все переходы с оборудованием |

> **Примечание по RES.DG (деградация / амортизация):** Амортизация оборудования — это экономическая категория; в физическом регистре она не существует напрямую. Физические прокси деградации — **η_pump** (снижение КПД) и **MTBF** (наработка на отказ, сут). Когда η_pump падает — это физический факт; Amort_pump в Bridge-формуле — это его экономическое следствие. Смешение: «амортизация растёт → это физическая проблема» является FM.EW.001.

> **Правило CC-B1.6.4:** Нельзя суммировать ресурсы разных типов без явной карты эквивалентностей. Электроэнергия и тепловая энергия суммируются через тепловой эквивалент (кВт·ч → Гкал: 1 Гкал = 1163 кВт·ч) только при явном указании этого коэффициента.

## 3. Физические параметры переходов конвейера нефти

Для каждого перехода — **что измеряется** и **полярность** (↓ лучше / ↑ лучше).

| Переход конвейера | Физический параметр | Символ | Единица | Полярность |
|------------------|--------------------|----|---------|------------|
| **Подъём жидкости** | Удельный расход электроэнергии | SW_lift | кВт·ч / т.жидк. | ↓ лучше |
| | КПД насосной установки (ЭЦН/ШГН) | η_pump | % | ↑ лучше |
| | Динамический уровень жидкости | DL | м | — (контекст) |
| | Газовый фактор | GOR | м³/т | — (контекст) |
| **Перекачка по трубопроводу** | Удельный расход электроэнергии | SW_pipe | кВт·ч / т·км | ↓ лучше |
| | Линейные потери давления | ΔP_lin | атм / км | ↓ лучше |
| **Подготовка нефти (УПН)** | Удельный расход реагентов | SW_chem | кг / т.жидк. | ↓ лучше |
| | Удельный расход тепла | SW_heat | Гкал / т.жидк. | ↓ лучше |
| | Остаточное содержание воды в нефти | Wres | % | ↓ лучше |
| **Весь конвейер** | Потери нефти (суммарные) | L_total | % от добычи | ↓ лучше |
| | Обводнённость продукции | WC | % | — (ключевой мультипликатор) |

## 4. Формулы-мосты (Bridge Formulas)

Явные формулы перевода физических параметров в экономические. Каждая формула версионируется отдельно от физических параметров.

### 4.1 Подъём жидкости → затраты на подъём нефти

```
OpEx_lift [руб/т.нефти] =
    SW_lift [кВт·ч/т.жидк.]
    × Tariff_EL [руб/кВт·ч]
    × (1 / (1 - WC))          ← пересчёт жидкость → нефть
    + Amort_pump [руб/т.нефти] ← амортизация насоса
```

**Ключевая переменная:** обводнённость `WC`.
При WC = 0%: коэфф. пересчёта = 1.0
При WC = 80%: коэфф. пересчёта = 5.0  ← в 5 раз дороже на т.нефти
При WC = 95%: коэфф. пересчёта = 20.0 ← в 20 раз дороже на т.нефти

**Следствие:** снижение обводнённости — главный рычаг управления затратами на подъём.

### 4.2 Перекачка → затраты на транспортировку нефти

```
OpEx_pipe [руб/т.нефти] =
    SW_pipe [кВт·ч/т·км]
    × Distance [км]
    × Tariff_EL [руб/кВт·ч]
    × (1 / (1 - WC))          ← если перекачивается жидкость, а не нефть
```

### 4.3 Подготовка нефти → затраты на подготовку

```
OpEx_proc [руб/т.нефти] =
    (SW_chem × Price_CH + SW_heat × Price_TH) [руб/т.жидк.]
    × (1 / (1 - WC))
    + Amort_UPN [руб/т.нефти]
```

### 4.4 Суммарные операционные затраты на конвейере нефти

```
OpEx_total [руб/т.нефти] =
    OpEx_lift + OpEx_pipe + OpEx_proc + OpEx_transport + ...
```

По **Δ_work (B.1.6)** — суммирование корректно, если все слагаемые в одной единице (руб/т.нефти) и переходы независимы (нет двойного счёта).

### 4.5 WLNK — Критические входные ресурсы по переходам (CC-B1.6.6)

По FPF CC-B1.6.6, критические входы Q* — это ресурсы, дефицит которых **полностью блокирует переход**, независимо от состояния остальных ресурсов. Для конвейера нефти критический ресурс определяет узкое место и приоритет оптимизации.

| Переход конвейера | Q* (критический ресурс) | Код | Что происходит при исчерпании Q* | Индикатор приближения к лимиту |
|------------------|------------------------|-----|----------------------------------|-------------------------------|
| **Подъём жидкости** | Электроэнергия | RES.EL | Насос останавливается → добыча = 0 | Нагрузка ЭЦН > 95% от номинала; перебои электроснабжения |
| | КПД оборудования (η_pump) | RES.DG | При η < 25% насос неэффективен → остановка или замена | η_pump < 35% = зона риска замены |
| **Перекачка** | Электроэнергия | RES.EL | Насосная станция останавливается → затоваривание | Нагрузка НС > 90% |
| | Пропускная способность трубопровода | — | При заполнении трубы — остановка добычи на куст | Q_pipe > 85% от проектной мощности |
| **Подготовка нефти (УПН)** | Деэмульгатор (реагент) | RES.CH | Нефть не достигает ГОСТ по содержанию воды → нельзя сдать | Остаток реагента < 3 сут. запаса |
| | Тепловая энергия | RES.TH | При T < минимума — деэмульсация неэффективна → Wres > ГОСТ | T теплоносителя < нормы |
| | Мощность УПН | RES.AM (физ.: η_UPN) | При перегрузке — рост Wres и/или остановка | Q_жидк. > 90% от проектной мощности УПН |
| **ППД (поддержание давления)** | Вода закачки | RES.WI | Закачка падает → давление пласта снижается → дебиты падают | Приемистость нагнетательных скважин падает |
| **Весь конвейер** | Обводнённость WC | — | При WC → 100%: всё Work_per_oil → ∞; операция нерентабельна | WC > 95% = экономический предел |

**Следствие для приоритизации:** При нескольких кандидатах на ГТМ — приоритет у ГТМ, затрагивающего Q* ограниченного перехода. Улучшение некритического ресурса при наличии блокирующего Q* не снимет ограничение (закон Голдратта о системных ограничениях).

## 5. Версионирование тарифов (Bridge Parameters)

Физические параметры (SW_lift, η_pump, WC) меняются при изменении физических условий или оборудования. Тарифы меняются независимо.

| Параметр | Код | Тип | Источник | Период применения |
|----------|-----|-----|----------|------------------|
| Тариф на электроэнергию | Tariff_EL | Bridge | Договор с поставщиком | Квартал / год |
| Цена химреагентов | Price_CH | Bridge | Договор с поставщиком | Квартал / год |
| Цена тепловой энергии | Price_TH | Bridge | Договор / норматив | Квартал / год |
| Норматив амортизации насоса | Amort_pump | Bridge | Норматив компании | Период регуляторного решения |
| Норматив амортизации УПН | Amort_UPN | Bridge | Норматив компании | Период регуляторного решения |

> **Правило версионирования:** При изменении тарифа — пересчитываются экономические показатели, физические остаются неизменными. Это позволяет разделять «мы стали эффективнее физически» и «тариф вырос».
