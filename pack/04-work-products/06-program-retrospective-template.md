# WP.RetroProgram: Шаблон ретроспективы программы инновационных проектов

**Назначение:** Канонический шаблон разбора программы проектов как организационной системы, а не просто набора отдельных НИР/НИОКР/ОПР.

**Версия:** 1.0  
**Статус:** Рабочий шаблон  
**Связано с:** [Моделью трех конвейеров](03-three-pipeline-model.md), [Ретроспективой инновационного проекта](05-innovation-project-retrospective-template.md)

---

## Принципы применения

1. Программа анализируется **поверх ретроспектив отдельных подпроектов**, а не вместо них.
2. Объект анализа - **тема, архитектура программы, зависимости и интегральный результат**.
3. Среднее арифметическое статусов подпроектов **не равно** статусу программы.

## Блок 2: Интегральный статус программы

| Статус | Когда применять |
|---|---|
| **Интегральный результат достигнут** | Программа дала подтвержденный результат по теме и перевела его во внедрение |
| **Частичный результат** | Часть темы закрыта, но контур неполон или эффект ограничен |
| **Получена карта знаний без рабочего контура** | Знание накоплено, но не собрано в рабочий контур решений |
| **Есть сильные отдельные проекты, но программа не сложилась** | Локальные успехи были, но архитектура программы слабая |
| **Программа организационно сорвалась** | Главный разрыв в ресурсах, ролях, интерфейсах, очередности |
| **Программа потеряла актуальность / изменилась стратегия** | Приоритет темы ушел и это зафиксировано осознанно |

## Блок 6: Главный разрыв программы и его причины

Выбрать один главный разрыв и описать его.

| Главный разрыв | Что это означает для программы |
|---|---|
| Выбор темы | Тема слишком широка, размыта или слабо привязана к производственной проблеме |
| Архитектура программы | Набор подпроектов неполон, дублируется или не образует контур |
| Очередность проектов | Поздние стадии запущены до готовности ранних |
| Управление зависимостями | Неописанные входы/выходы, нет владельца стыка |
| Ресурсы и внимание | Конкуренция за полигоны, бюджет, экспертизу |
| Управленческий контур | Нет ритма решений, ролей, gate-логики |
| Внедрение и тиражирование | Контур знания не переведен в эксплуатацию |
| Измерение интегрального эффекта | Программа не может показать общий результат |
| Актуальность / смена стратегии | Внешний контур изменился быстрее программы |

## Блок 8: Решение по следующему циклу

| Решение | Когда выбирать | Выбрано |

| Закрыть программу | Тема исчерпана или потеряла актуальность | |

## Минимальная spreadsheet-композиция

Если шаблон реализуется в таблице, рекомендуемый состав листов такой:

| Лист | Назначение |
|---|---|
| **Инструкция** | Краткий порядок работы с шаблоном |
| **Справочники** | Выпадающие значения по статусам, разрывам, зависимостям, решениям |
| **Классификатор_причин** | Причины на уровне программы |
| **Реестр_программ** | Сводная таблица по программам |
| **Реестр_подпроектов** | Фактология по подпроектам |
| **Карта_покрытия** | Проверка дыр и дублей |
| **Зависимости_и_стыки** | Проверка handover между подпроектами |
| **5_Почему** | Разбор ключевого разрыва |
| **Форма_программы** | Подробная форма на одну программу |
| **Дашборд** | Сводка по статусам, разрывам и уровню внедрения |
