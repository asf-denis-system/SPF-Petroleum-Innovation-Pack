# WP.ThreePipelines: Модель трёх конвейеров

**Назначение:** Управляемая модель трёх взаимосвязанных конвейеров upstream-производства — нефти, проектов и технологий. Позволяет видеть где застревает нефть/проект/технология и кто отвечает за каждый переход.

**Версия:** 1.0
**Статус:** Рабочая модель
**Связано с:** [Методология внедрения инноваций](01-innovation-implementation-methodology.md), [Шаблон инновационного проекта](02-innovation-project-template.md)

---

## Как три конвейера стыкуются

```
Конвейер НЕФТИ           ←  место застревания  →  БВ/ТВ
   (операционный)                                     ↓
                                               Конвейер ПРОЕКТОВ
                                                  (управленческий)
                                                     ↓
                                               Конвейер ТЕХНОЛОГИЙ
                                                  (R&D/НИОКР)
                                                     ↓
                                            Готовая технология → обратно в нефть
```

1. **Конвейер нефти** выявляет место застревания → формируется БВ/ТВ на переходе.
2. **Конвейер проектов** берёт БВ/ТВ → выбирает способ закрыть → организует финансирование/объект/решения/тиражирование.
3. **Конвейер технологий** внутри проекта «дозревает» метод до уровня, когда его можно испытать и тиражировать.

## 1. Шаблон: Конвейер целевой системы «НЕФТЬ» (операционный поток)

**Цель конвейера:** Непрерывно переводить нефть из «потенциала в пласте» в «стабильную поставку на поверхность/товар», с контролем узких мест.

**Что движется по конвейеру (объект):** Поток нефти (запасы/дебит/тонны) + «доступность» (можно ли извлечь/довести).

**Границы:** От геологии/плана до товарной подготовки.

### Стадии, выходы и контроль качества (Definition of Done)

| Стадия (состояние) | Цель стадии | Выход (что должно быть готово) | Проверка / критерий | Типовые блокеры | Владелец стыка |
|---|---|---|---|---|---|
| Запланирована к добыче | Выбрать приоритетные объёмы/зоны | План/портфель объектов, целевые KPI | План согласован, есть baseline (текущие потери) | Нет единого приоритета | Владелец портфеля/разработки |
| Происследована | Понять свойства/неопределённости | Отчёты, данные, обновлённые модели | Параметры достаточны для решения «где бурить/как воздействовать» | Данных мало/разрознены | Геология + моделирование |
| Локализована | «Где именно нефть» и сколько | Карты/зоны, перечень запасов/рисков | Точность/достоверность достаточна для капвложений | Высокая неопределённость | Геология + разработка |
| Подготовлена к добыче | Подготовить решение по доступу | Проект бурения/заканчивание/инфра | Есть ТЭО, риски, ресурсы | Капекс/инфра ограничения | Разработка + строительство |
| Открыт доступ | Физически обеспечить доступ | Скважины/инфра/готовность фонда | Выполнены техусловия, готовность к эксплуатации | Задержки бурения/МТО | Бурение/промысел |
| Создано воздействие (ППД/МУН) | Обеспечить приток/охват | Проект/режимы, готовность закачки | Охват/приемистость в норме, контролируется | Разрыв «пласт–наземка» | Промысел + разработка |
| Поднята на поверхность | Устойчиво добывать | Режимы, оборудование, контроль осложнений | Дебит/обводнённость/простой в рамках KPI | Осложнения, аварийность | Промысел |
| Подготовлена/доставлена | Довести до товарного качества | Подготовка, транспорт, качество | Качество/потери/ограничения в норме | Узкие места сети | Транспорт/подготовка |

### Метрики управления (минимум)

- **Throughput:** т/сут, т/мес; % выполнения плана.
- **Lead time по ключевым переходам:** «локализация→доступ», «доступ→устойчивый дебит».
- **WIP/очередь:** сколько объектов «зависло» на стадии > N недель.
- **Качество:** обводнённость, простои, аварии, отклонение факта от модели.

### Стоп-сигналы

- «Зависли» решения на стыке > SLA.
- Эффект/дебит не подтверждается данными мониторинга.
- Повторяющиеся возвраты к прежним стадиям без устранения причины.

### Стыки с другими конвейерами

- Проблемы/узкие места на переходах → становятся входом в конвейер проектов (БВ/ТВ).
- Любая «новая методика/оборудование/реагент» → вход в конвейер технологий.

## 2. Шаблон: Конвейер проектов (от БВ/ТВ до тиражирования)

**Цель конвейера:** Превращать бизнес/технологические проблемы промысла во внедрённые решения с подтверждённым эффектом и масштабированием.

**Что движется:** Проект (как контейнер изменения) + его статус, риск и эффект.

**Границы:** От регистрации БВ/ТВ до тиражирования/закрепления в регламентах.

### Стадии, выходы и критерии

| Стадия | Выход (артефакт) | Критерий «готово» | Решение на воротах | Владелец |
|---|---|---|---|---|
| 1. Регистрация БВ/ТВ | Карточка БВ/ТВ + привязка к участку конвейера «НЕФТЬ» | Ясно «что болит», где болит, baseline потерь | GO в оценку | Владелец участка/ПЭО |
| 2. Приоритизация | Рейтинг/портфель + причина приоритета | Критерии понятны (эффект/риск/срок/масштаб) | GO в подбор решения | Портфельный комитет |
| 3. Подбор решений | Список вариантов + критерии выбора | Есть сравнение вариантов (почему этот) | GO в инициацию | Техлид + бизнес |
| 4. Инициация проекта | Паспорт, объект(ы), план, бюджет, риски | Есть ресурсы, ответственные, план испытаний | GO в реализацию | PM/владелец конвейера |
| 5. Реализация / разработка | «Версия решения» + готовность к испытанию | Готово к тесту безопасно и повторяемо | GO в ОПИ/ОПР | PM + техлид |
| 6. Испытания (ОПИ/ОПР) | Протокол/отчёт + данные мониторинга | Эффект измерим, границы применимости понятны | GO / NO-GO / PIVOT | Владелец испытаний |
| 7. Упаковка внедрения | ИЛП/пакет внедрения (инструкции, требования, обучение) | Можно тиражировать без «героев» | GO в тиражирование | Продукт-оунер технологии |
| 8. Тиражирование | План масштабирования + список подходящих объектов | Назначены владельцы внедрения, контроль эффекта | Закрытие/перевод в эксплуатацию | Операционный владелец |

### Метрики конвейера проектов

- Время прохождения стадий (особенно «приоритизация→инициация», «испытания→решение»).
- WIP по стадиям (сколько проектов «в очереди»).
- Доля проектов с подтверждённым эффектом и доля, дошедшая до тиражирования.
- Возвраты/переподборы (сколько раз проект откатывался назад и почему).

### Стоп-сигналы / анти-паттерны

- Нет baseline → нечем доказать эффект.
- Испытание без заранее согласованного метода измерения эффекта.
- «Технология сделана», но нет пакета внедрения/тиражирования.

### Стыки

- Вход: проблемы на переходах конвейера нефти.
- Внутри проекта: технологическая часть идёт по конвейеру технологий.

## 3. Шаблон: Конвейер технологий (развитие технологии как продукта)

**Цель конвейера:** Довести метод/технологию от идеи до состояния «можно безопасно испытать и повторяемо внедрять».

**Что движется:** Технология/метод как продукт + его зрелость (TRL/готовность к испытанию/внедрению).

**Границы:** От концепции до подтверждённого результата и пакета для внедрения.

### Стадии технологии и критерии

| Стадия | Выход (что должно быть) | Проверка / критерий качества | Типовые риски | Владелец |
|---|---|---|---|---|
| 1. Концепция | Визитка/ТЗ: для какой БВ/ТВ, где применимо | Понятно «зачем/где/какой эффект/ограничения» | «Идея без объекта» | Техлид/продукт |
| 2. Обоснование/модель | Расчёты/моделирование/логика механизма | Есть гипотезы и диапазоны применимости | Модель не бьётся с реальностью | R&D |
| 3. Прототип (лаб/стенд) | Прототип + план испытаний | Воспроизводимость, безопасность, критерии успеха | «Работает один раз» | R&D + QA |
| 4. Образец для объекта | Комплект для ОПИ (материалы/оборудование/регламенты) | Готовность к полевым условиям, МТО/логистика | Не готово «в поле» | R&D + производство |
| 5. Испытание на объекте | Протокол, данные, выводы | Эффект подтверждён по методике, понятны границы | Неверная методика измерения | Владелец испытаний |
| 6. Упаковка продукта | ИЛП: инструкции, требования, обучение, сервис | Можно повторить без разработчиков | «Не тиражируется» | Продукт-оунер |
| 7. Поддержка и улучшения | Версии технологии, база кейсов | Есть обратная связь и цикл улучшения | Деградация качества | Продукт + эксплуатация |

### Метрики технологии

- Время «идея→готово к ОПИ», «ОПИ→готово к тиражированию».
- Доля успешных испытаний (с объяснением причин неуспеха).
- Повторяемость эффекта (вариативность по объектам).
- Стоимость/сложность внедрения (TCO), требования к персоналу/сервису.

### Стоп-сигналы

- Нет чётких границ применимости.
- Нельзя измерить эффект «по-честному» (нет методики/данных).
- Для внедрения нужны «герои» (знание не упаковано).

### Стыки

- Вход: запрос из конвейера проектов (под конкретную БВ/ТВ).
- Выход: «готовая к внедрению технология» обратно в проект и дальше в эксплуатацию (конвейер нефти).

## Артефакты по умолчанию

| Артефакт | Конвейер | Назначение |
|---|---|---|
| Карточка БВ/ТВ | Проектов | Регистрация проблемы с привязкой к переходу нефти и baseline потерь |
| Паспорт проекта | Проектов | Объект испытаний, план, ресурсы, риски, путь на УК |
| План ОПИ/ОПР | Проектов + технологий | Метод измерения эффекта, критерии GO/NO-GO, команда |
| Протокол эффекта | Проектов + технологий | Факт vs план, границы применимости, выводы |
| ИЛП (инструкция по ликвидации проблемы) | Технологий | Инструкция, требования к сервису/подрядчику, обучение, регламенты |
