# [UP.M.010] Верификация второго принципа на новых данных

## Определение

**Метод** — регулярная проверка второго принципа (OA.PP.XXX, OA.SP.YYY и т.д.) на основе новых натурных данных. Если наблюдаемое поведение соответствует принципу → принцип верен. Если нет → либо данные неправильные, либо принцип нужно переинтерпретировать/переформулировать.

## Цель

1. **Проверить** второй принцип в условиях нашего месторождения
2. **Обнаружить** исключения и граничные случаи (где принцип не работает?)
3. **Уточнить** формулировку принципа
4. **Собрать фактический материал** для развития знания (мирового и нашего)

## Входные данные

| Входные данные | Описание |
|---|---|
| **Второй принцип** | Например: "Энергия конечна" (OA.PP.001) |
| **Натурные данные** | Давление, добыча, состав жидкостей за N лет (UP.M.005, UP.M.006) |
| **Сценарии/модели** | Что предсказывала модель (основанная на принципе)? |
| **Исторические события** | Что на самом деле случилось? |

## Выходные данные

| Рабочий продукт | Описание |

| **WP.Verification.001** | Таблица: принцип → предсказание → реальность → совпадение? |

## Процесс

### Этап 1: Выбрать принцип для проверки (3–5 дней)

Обычно проверяем принципы, которые:
- Направляют ключевые решения (ППД, режимы скважин)
- Вызывают сомнения (кто-то в команде не верит?)
- Новых для компании (внедрили из этого pack'а)

**Примеры:**
- **OA.PP.001** (Энергия конечна) — теоретическая основа для ППД; нужно проверить
- **OA.PP.003** (Пласт гетерогенен) — видно ли это в дебитах скважин?
- **OA.SP.002** (Запаздывание реакции) — через какое время пласт реагирует на вмешательство?

### Этап 2: Сформулировать проверяемую гипотезу (1 неделя)

На основе второго принципа написать **предсказание** в терминах наблюдаемых переменных.

**Пример 1: OA.PP.001 (Энергия конечна)**
- Принцип в словах: "Пластовая энергия исчерпаема; без поддержки давления месторождение будет деградировать"
- Предсказание: "Если не будет ППД, то давление упадёт с темпом ~1 МПа/год, дебит упадёт экспоненциально, КИН ≤ 35%"
- Проверяемые переменные: давление(t), дебит(t), совокупная добыча, КИН

**Пример 2: OA.SP.002 (Запаздывание реакции)**
- Принцип: "Пласт реагирует на вмешательства с задержкой (недели-месяцы)"
- Предсказание: "Если закрыть скважину на дебит в день 0, то соседи упадут в дебите в день 10–30, не сразу"
- Проверяемые переменные: дебиты соседей vs дата воздействия (cross-correlation)

### Этап 3: Собрать и подготовить данные (2–4 недели)

Извлечь из SCADA/базы данных:
- Ежедневные дебиты и давления за весь период разработки (или последние 5 лет)
- События: когда были ГРП, когда запустили ППД, когда отказали скважины
- Качество данных: пропуски, выбросы, есть ли ошибки измерений?

### Этап 4: Статистический анализ (2–3 недели)

**Для OA.PP.001 (Энергия):**
1. Вычислить темп падения давления (∆P/∆t) отдельно для периодов:
   - До ППД включения
   - При включённой ППД
2. Сравнить с предсказанием модели
3. Нарисовать график: давление vs время, наложить предсказание

**Для OA.SP.002 (Запаздывание):**
1. Найти события (закрытие скважины, изменение режима)
2. Для каждого события вычислить cross-correlation между возмущением и ответом в соседних скважин
3. Определить лаг (delay) — на сколько дней соседи отреагировали

**Для OA.PP.003 (Гетерогенность):**
1. Собрать дебиты всех скважин в один месяц
2. Вычислить разброс (min, max, среднее квадратичное отклонение)
3. Нарисовать гистограмму: есть ли разброс > 50%?

### Этап 5: Сравнить предсказание vs реальность (1–2 недели)

Построить таблицу:

| Принцип | Предсказание модели | Реальное наблюдение | Абсолютная ошибка | Совпадение? |
|---|---|---|---|---|
| **OA.PP.001** | Темп падения 1.0 МПа/год | Наблюдалось 0.95 МПа/год | 5% | ✓ Хорошо |
| **OA.SP.002** | Лаг 15 дней | Наблюдалось 12–18 дней | < 20% | ✓ Хорошо |
|  **OA.PP.003** | Разброс дебитов > 30% | Наблюдалось 25–35% | < 10% | ✓ Хорошо |

### Этап 6: Анализ отклонений (если есть) (1 неделя)

Если ошибка > 20%:
- **Версия 1:** Данные неправильные (ошибка калибровки, пропуски)
- **Версия 2:** Модель неправильная (например, неправильный ввод)
- **Версия 3:** Принцип нужна переинтерпретировать (he просто "энергия конечна", а "энергия конечна при стабильной геологии; если геология меняется, энергия может восстановиться")

### Этап 7: Уточнить формулировку принципа (если нужно) (1–2 недели)

Если хап обнаружены, обсудить с командой:
- Формулировка старая: "Энергия конечна"
- Новое понимание: "Энергия конечна при исходном состоянии геологии; перемещение или трещины могут изменить доступную энергию"
- Границы применимости: "Принцип верен для горизонтов глубины 2000–3500 м, при температурах 30–60°C, для пластов с проницаемостью > 10 мД"

### Этап 8: Документировать и делиться (1–2 недели)

Написать доклад:
- Какой принцип проверяли
- Что предсказывали
- Что наблюдали
- Заключение: верен ли принцип? На 100% или с оговорками?
- Уроки для улучшения модели

Поделиться результатами:
- Внутри компании (обязательно!)
- В журналах/конференциях (если интересно научному сообществу)

## Пример

**Месторождение Восток, проверка OA.PP.001 (Энергия конечна)**

**Предсказание (из модели):**
- Темп падения давления без ППД: 1.0 МПа/год
- С ППД: давление стабилизируется
- КИН без ППД: ~30%; с ППД: ~40%

**Натурные данные (10 лет разработки):**
- Летопись 1–3: без ППД, давление упало с 45 на 42 МПа (3 МПа за 3 года = 1.0 МПа/год) ✓
- Год 4: включена ППД, давление захватывает и стабилизируется на 33 МПа ✓
- Год  5–10: давление держится стабильно ±2 МПа ✓
- Совокупная добыча (10 лет): 8.5 млн т нефти (КИН = 38%) ✓

**Анализ:**
- Ошибка предсказания: < 5% по всем параметрам
- Принцип OA.PP.001 подтверждён ✓

**Вывод:** Принцип "Энергия конечна" полностью верен в условиях нашего месторождения. Рекомендация для других месторождений: используйте этот принцип как основу для стратегии ППД.

## Когда метод не работает

- **Слишком мало данных** → статистика ненадëжна
- **Несколько факторов одновременно** → трудно изолировать причину (какой фактор виноват?)
- **Принцип очень общий** → трудно сформулировать проверяемую гипотезу
