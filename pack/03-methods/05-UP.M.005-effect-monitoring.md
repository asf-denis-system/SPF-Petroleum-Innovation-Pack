# [UP.M.005] Мониторинг эффекта вмешательства на основе натурных данных

## Определение

**Метод** — сбор, анализ и интерпретация фактических данных о результатах техвмешательств (гидравлический разрыв пласта, кислотная обработка, изменение режима скважин) на основе измерений давлений, дебитов, состава продукции до и после работ.

## Цель

1. **Измерить** фактический эффект вмешательства (насколько поднялся дебит?)
2. **Понять механизм**: почему изменился дебит? (трещина открыта, проводимость увеличена, кожа-фактор поднялся?)
3. **Сравнить** прогноз vs реальность (модель предсказала +50%, а реально +30% → модель переоценивает?)
4. **Собрать данные** для верификации второго принципа (UP.M.010)

## Входные данные

| Входные данные | Описание |
|---|---|
| **История работ** | Дата, описание работы (ГРП, кислота), параметры (объём жидкости, давление) |
| **Дебитомеры** | Суточные объёмы нефти, воды, газа до и после работы |
| **Манометры** | Давления в скважинах до и после работы |
| **Анализы жидкостей** | Лабораторные анализы нефти, воды, газовых проб |
| **Прогноз** | Что ожидали от работы (из модели или инженерного опыта) |

## Выходные данные

| Рабочий продукт | Описание |
|---|---|
| **WP.Monitor.001** | Таблица: все работы, их результаты (дебит до/после, прирост) |
| **WP.Monitor.002** | Анализ механизмов эффекта (кожа-фактор, проводимость, заложенность) |
| **WP.Monitor.003** | Сравнение прогноз vs факт (график рассеяния, средняя ошибка) |
| **WP.Monitor.004** | Оценка надёжности и воспроизводимости метода (какой % работ имеет успех?) |
| **Доклад** | Выводы, рекомендации по улучшению техвмешательств |

## Процесс

### Этап 1: Стандартизация данных (1 неделя)

Все скважины должны измеряться одинаково:
- **Дебитомер:** единая методология откалибрована один раз, затем не меняется
- **Манометр:** электронный, точность ±1%, снимаются ежедневно
- **Анализы:** одна лаборатория, одна методология

### Этап 2: Сбор базовых данных до работы (2–3 недели)

Перед вмешательством установить стабильный режим:
- Скважина работает 2–3 недели без изменений
- Каждый день снимаются: дебит нефти, воды, газа; давления на устье и перед пластом
- Вычислить среднее и стандартное отклонение (baseline)

### Этап 3: Выполнить работу и начать мониторинг (1–2 дня)

После вмешательства (ГРП, кислота):
- Скважина перекрывается на восстановление давления (24–72 часа в зависимости от типа)
- Затем открывается и выходит на стабильный режим (обычно 1–2 недели с постепенным ростом дебита)

Во время восстановления и выхода на режим — ежедневный мониторинг.

### Этап 4: Сбор данных после работы (3–4 недели)

После 3–4 недель стабилизации:
- Вычислить среднее значение дебита, давления (post-work baseline)
- Вычислить прирост: $\Delta Q = (Q_{\text{post}} - Q_{\text{pre}}) / Q_{\text{pre}} \times 100\%$

### Этап 5: Техническая интерпретация (2–3 недели)

На основе давлениетеста после работы вычислить:
- **Кожа-фактор** (skin factor) — сопротивление около скважины
  - До работы: skin = +3 (засорение, осложнения)
  - После ГРП: skin = -2 (трещина помогает)
  - ∆skin = -5 → прирост проведения трещины
- **Проводимость трещины** (fracture conductivity) — как хорошо трещина проводит жидкость?

### Этап 6: Сравнение прогноз vs факт (1 неделя)

Для каждой работы:
- Что предсказала модель (до проведения работы)? ∆Q_pred = +50%
- Что получилось на самом деле? ∆Q_fact = +35%
- Отклонение: (∆Q_fact - ∆Q_pred) / ∆Q_pred = -30% (модель переоценивала)

Построить график на все работы (X = predicted, Y = actual):
- Если точки на диагонали → модель хорошая
- Если точки выше диагонали → модель недооценивает (хорошо!)
- Если точки ниже диагонали → модель переоценивает (плохо, нужна калибровка)

### Этап 7: Статистика (1 неделя)

Для каждого типа работы вычислить:
- **Средний прирост** (e.g. ГРП в среднем даёт +45% дебита)
- **Стандартное отклонение** (разброс от -10% до +80%?)
- **Процент неудач** (сколько работ дали < 10% прироста?)
- **ROI** (затраты на работу vs прирост дохода)

## Инструменты

| Инструмент | Для чего |
|---|---|
| **Pressure transducers** | Электронные манометры для ежедневного мониторинга |
| **Flow meters** | Калибр. дебитомеры (ЧОП, ротаметры) |
| **Well interaction test analysis** | Типизация analysis — интерпретация давлениетестов |
| **Excel / Python** | Статистика, графики, сравнение |

## Пример

**Скважина W-7, ГРП проведён в марте 2025**

**До работы:**
- Средний дебит: 25 т/день
- Давление на забое: 18 МПа
- Skin-фактор: +2.5

**После работы (апрель):**
- Средний дебит: 38 т/день (прирост +52%)
- Давление на забое: 17 МПа (практически не изменилось)
- Skin-фактор: -1.0
- ∆skin = -3.5

**Прогноз был:** +40% (была удовлетворена)
**Факт:** +52% (лучше прогноза на 30%)

**Вывод:** ГРП очень удачная, трещина хорошая, skin снизился на 3.5 — почти идеально.

## Когда этот метод не работает

- **Плохая калибровка дебитомеров** → ошибки в измерениях
- **Летучесть скважины** (дебит скачет день в день) → сложно выявить тренд
- **Одновременные события** (в один день ГРП на W-7 и отказ на W-8) → непойму что на что влияет
